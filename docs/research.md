3. **OGQ(네이버)**는 AI 생성 콘텐츠를 **허용**하고 오히려 장려
4. **카카오**는 AI 이모티콘 입점을 **제한** (2023~현재, 핵심 리스크)

## 2. 기술 스택 분석

### 2.1 AI 이미지 생성 모델 비교

| 모델 | 얼굴 유사도 | 스티커 품질 | LoRA/커스텀 | 속도 (1장) | 라이선스 | 추천도 |
|------|-----------|-----------|-----------|----------|---------|-------|
| **SDXL + IP-Adapter-FaceID** | ★★★★☆ | ★★★★★ | 풍부 (스티커 LoRA 다수) | ~3-5초 (L40S) | Open | **1순위 (자체 서버)** |
| **Flux + IP-Adapter** | ★★★★★ | ★★★★☆ | 부족 (스티커 LoRA 미흡) | ~5-8초 | 제한적 | 3순위 (성숙 후 전환) |
| **fal.ai Face-to-Sticker API** | ★★★★☆ | ★★★★☆ | 불가 (고정) | ~3-5초 | API 종량제 | **2순위 (MVP/PoC)** |
| **Replicate SDXL API** | ★★★☆☆ | ★★★☆☆ | 커스텀 모델 배포 가능 | ~5-10초 | API 종량제 | 4순위 |

**추천 전략**:
- **MVP/PoC**: fal.ai Face-to-Sticker API로 빠르게 검증 (~$0.03~0.04/이미지)
- **정식 서비스**: SDXL + IP-Adapter-FaceID + 스티커 LoRA 자체 서버 구축
- **향후**: Flux 생태계 성숙 시 전환 검토

### 2.2 핵심 기술 컴포넌트

| 컴포넌트 | 기술 | 용도 | 비고 |
|---------|------|------|------|
| **얼굴 특징 추출** | InsightFace / ArcFace | 사진에서 얼굴 임베딩 추출 | IP-Adapter-FaceID 내장 |
| **스타일 변환** | IP-Adapter-FaceID-PlusV2 + SDXL | 얼굴 유지하면서 스타일 적용 | [HuggingFace](https://huggingface.co/h94/IP-Adapter-FaceID) |
| **표정 제어** | ControlNet (OpenPose/MediaPipe) | 다양한 표정/포즈 프리셋 적용 | 24~32개 표정 세트 필요 |
| **배경 제거** | rembg / SAM2 | 투명 배경 PNG 생성 | 카카오톡 규격 필수 |
| **이미지 후처리** | Pillow / Sharp | 360x360 리사이징, PNG 최적화 | 규격 맞춤 자동화 |
| **텍스트 오버레이** | Canvas API / Pillow | 한글 텍스트/말풍선 추가 | 한글 폰트 번들 필요 |

## 5. 개인정보/법적 검토

### 5.1 얼굴 사진 = 생체정보 (한국 PIPA)

| 항목 | 요구사항 |
|------|---------|
| **개인정보보호법 (PIPA)** | 얼굴 사진은 민감정보(생체정보)로 분류 |
| **수집 동의** | 명시적 동의 필수 (수집 목적, 보관 기간, 처리 방법) |
| **보관 제한** | 이용 목적 달성 후 즉시 삭제 권장 |
| **해외 전송** | GPU 서버가 해외에 있으면 개인정보 국외 이전 동의 필요 |
| **PIPC 강화 동향** | 생체정보 규제 프레임워크 강화 추진 중 (2024~) |

### 5.2 AI 저작권

| 항목 | 현황 |
|------|------|
| AI 생성물 저작권 | 한국: 명확한 법률 없음 (판례 형성 중) |
| AI 이모티콘 판매 가능성 | OGQ: 가능 (UCI 발급), 카카오: 불가 (입점 제한) |
| 타인 얼굴 도용 위험 | 본인 사진 인증 + 동의 절차 필수 |
| 초상권 이슈 | 유명인 사진 차단 메커니즘 필요 |

### 5.3 법적 리스크 대응

1. **개인정보 처리방침** 작성 (전문 법률 자문 권장)
2. **사진 즉시 삭제 정책**: 이모티콘 생성 후 원본 사진 삭제
3. **본인 인증**: 셀카 촬영 시 liveness detection 고려
4. **이용약관**: AI 생성물의 저작권 귀속, 사용 범위 명확히
5. **GPU 서버 위치**: 가능하면 한국 리전 사용 (개인정보 국외 이전 회피)

## 7. 종합 의견

### 7.1 기회

1. **블루오션**: 한국 시장에서 얼굴 기반 AI 이모티콘 생성 서비스가 없음
2. **기술 준비도**: IP-Adapter-FaceID, ControlNet 등 핵심 기술이 이미 성숙
3. **시장 규모**: 한국 이모티콘 시장 1.5조원, 월 3천만 사용자
4. **OGQ(네이버) 허용**: AI 생성 콘텐츠 허용 + UCI 발급 → 즉시 판매 가능
5. **비용 효율**: 자체 서버 시 세트당 $0.05~0.10, 높은 마진 가능

### 7.2 리스크

1. **🔴 카카오톡 AI 입점 제한**: 핵심 차별점이었던 카카오톡 이모티콘 승인 연동이 현재 불가능. 정책 변경 시점 불명확
2. **🟡 얼굴 일관성**: 24~32장의 표정/포즈에서 동일인물 유사도 유지가 기술적 도전
3. **🟡 개인정보 규제**: 얼굴 사진은 생체정보 → PIPA 준수, 법률 자문 필요
4. **🟡 Mirror AI 경쟁**: YC 배치 + $3.5M 투자. 다만 한국 시장 미진출
5. **🟢 GPU 비용**: 스케일링 시 비용 관리 필요하나, 마진 구조는 양호

### 7.3 전략 제안: 피봇 필요

**원래 핵심 차별점**: 카카오톡 이모티콘 승인 연동
**현실**: 카카오톡 AI 이모티콘 입점 제한 (2023~현재)

**수정 전략 제안**:

| 단계 | 전략 | 이유 |
|------|------|------|
| **v1.0** | **개인 사용 + OGQ 마켓 중심** | 카톡 이모티콘 판매는 안 되지만, 카톡/SNS에서 개인 사용은 제한 없음. OGQ에서는 AI 판매 가능 |
| **v1.1** | **작가 도구 + 마켓플레이스** | AI 보조 도구로 작가 지망생 타겟. 생성 후 수동 수정 → 카카오 제출 (AI '보조' 포지셔닝) |
| **v1.5** | **카카오 정책 변경 대비** | 카카오톡 규격 맞춤 기능 사전 구현, 정책 변경 즉시 대응 |
| **v2.0** | **B2B 브랜드 이모티콘** | 브랜드 이모티콘은 별도 심사 체계 → AI 활용 가능성 높음 |
