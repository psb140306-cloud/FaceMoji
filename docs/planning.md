## 1. 프로젝트 개요

### 1.1 배경 및 목적

한국 이모티콘 시장은 약 1.5조원 규모이며, 월 3,000만 명이 이모티콘을 사용한다. 그러나 이모티콘 제작은 그림 실력이 필수이고, 외주 비용은 50만원 이상, 직접 제작 시 수주~수개월이 소요된다.

AI 이미지 생성 기술(IP-Adapter-FaceID, ControlNet)이 성숙 단계에 진입했으나, **한국 시장에 특화된 얼굴 기반 AI 이모티콘 생성 서비스는 아직 없다**. 특히 네이버 OGQ 마켓은 AI 생성 콘텐츠를 공식 허용하고 UCI(국가콘텐츠식별체계) 발급까지 제공하여 즉시 판매 가능한 합법적 수익 경로가 존재한다.

**핵심 기회**: 한국 시장 + 얼굴 기반 AI + OGQ 규격 특화 = 블루오션

### 1.2 목표

**핵심 목표:**
- 셀카 1장으로 24개 이모티콘 세트를 자동 생성하는 웹 서비스 구축
- 네이버 OGQ 마켓 규격(740×640px, 24개 세트)으로 자동 변환하여 바로 판매 가능하게 함
- 자체 마켓플레이스를 통한 크리에이터 이코노미 생태계 구축

**부가 목표:**
- 카카오톡 AI 입점 정책 변경 시 즉시 대응 가능한 규격 맞춤 기능 사전 구현
- B2B 브랜드 이모티콘 서비스로 확장
- 반려동물/커플/가족 이모티콘 등 파생 카테고리 개척

### 1.3 타겟 사용자

| 세그먼트 | 설명 | 니즈 | 우선순위 |
|---------|------|------|---------|
| MZ세대 일반 사용자 | 카톡/인스타에서 나만의 이모티콘을 쓰고 싶은 10~30대 | 쉬운 생성, 재미, 공유 | 1순위 (MVP) |
| 이모티콘 작가 지망생 | 그림 실력이 부족하지만 이모티콘 제작에 관심 | AI 보조 제작, OGQ 제출 | 2순위 (v1.0) |
| 기업/브랜드 | 브랜드 캐릭터 이모티콘을 만들고 싶은 소상공인/기업 | 빠른 브랜드 이모티콘, 저비용 | 3순위 (v2.0) |

**핵심 사용 시나리오:**
1. 사용자가 셀카를 업로드 → 스타일 선택 → 3분 내 24개 이모티콘 세트 생성
2. OGQ 규격으로 자동 변환 → 다운로드 → OGQ 크리에이터 스튜디오에 제출
3. 생성한 이모티콘을 카톡/인스타 프로필에 개인 사용
4. 자체 마켓플레이스에서 다른 사용자에게 판매

## 3. 기술 방향

### 3.1 아키텍처 개요

```
사용자 (브라우저)
    ↕ HTTPS
Next.js 프론트엔드 (Vercel)
    ↕ API Routes / tRPC
FastAPI 백엔드 (AI 서버)
    ↕ RunPod Serverless / fal.ai API
AI 파이프라인 (SDXL + IP-Adapter-FaceID + ControlNet + rembg)
    ↕
Supabase (Auth + DB + Storage)
    ↕
Toss Payments (결제)
```

### 3.2 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| **프론트엔드** | Next.js 15 + React 19 + TypeScript | SSR/SEO, 기존 스택, Vercel 배포 |
| **UI** | Tailwind CSS + shadcn/ui | 빠른 UI 개발, 일관된 디자인 |
| **AI 백엔드** | FastAPI (Python) | AI 모델 연동(Python 생태계), GPU 서버 분리 |
| **AI 모델 (MVP)** | fal.ai Face-to-Sticker API | 빠른 검증, 배경 제거 자동 포함, ~$0.02~0.04/이미지 |
| **AI 모델 (정식)** | SDXL + IP-Adapter-FaceID + ControlNet | 자체 서버, 높은 커스터마이징, 세트당 ~$0.05~0.10 |
| **GPU 서버** | RunPod Serverless (L40S) | $1.33/hr, 서버리스 스케일링 |
| **DB** | Supabase (PostgreSQL) | Auth 내장, Storage, 기존 경험 |
| **파일 저장** | Supabase Storage | 이모티콘 이미지 저장, CDN |
| **결제** | Toss Payments | 한국 원화 결제, PG 수수료 2.5~3.5% |
| **배포** | Vercel (웹) + RunPod (AI) | 분리 아키텍처, 비용 효율 |
| **모니터링** | Sentry + Vercel Analytics | 에러 추적, 사용자 분석 |

### 3.3 AI 파이프라인 상세

```
[셀카 업로드]
    ↓
[얼굴 감지 + 품질 검증] — InsightFace
    ↓
[얼굴 임베딩 추출] — ArcFace (IP-Adapter-FaceID 내장)
    ↓
[24개 표정 프리셋 루프]
    ├→ [ControlNet 포즈 적용] — OpenPose/MediaPipe 프리셋
    ├→ [스타일 LoRA 적용] — 카툰/플랫/애니메 등
    └→ [SDXL + IP-Adapter 생성] — instant_id_strength: 0.7
    ↓
[배경 제거] — rembg / SAM2
    ↓
[리사이징 + PNG 최적화] — 740×640px (OGQ) / 360×360px (카카오 예비)
    ↓
[세트 패키징] — ZIP 다운로드 (스티커 24개 + 메인 + 탭)
```

## 5. 제약사항 및 고려사항

### 5.1 기술적 제약
- **얼굴 일관성**: 24개 표정에서 동일인물 유사도 유지가 핵심 기술 도전
- **OGQ API 미제공**: 수동 업로드만 가능 → 규격 변환 + 가이드 PDF 방식으로 대응
- **fal.ai → RunPod 전환**: 월 500세트 초과 시 자체 서버 전환 필요

### 5.2 법적 제약
- **PIPA 생체정보**: 얼굴 사진 = 민감정보, 명시적 동의 + 즉시 삭제 필수
- **국외 이전**: fal.ai/RunPod 서버가 해외 → 개인정보 국외 이전 동의 필요
- **AI 저작권**: 한국에서 AI 생성물 저작권 미확립, 이용약관에 명시
- **카카오 AI 입점 제한**: 2023~현재 유지 중, OGQ 우선 전략으로 대응

### 5.3 비용 제약
- 초기 투자: ~$200~500 (도메인, 법률 자문, Supabase Pro)
- 월 운영비: ~$100~1,000 (트래픽에 따라 변동)
- 법률 자문: $200~500 (개인정보 처리방침, 이용약관)
